# Privacy Principles

# BayWel Privacy Principles

# Core Principle

User trust is more valuable than aggressive data collection.

BayWel should minimize surveillance behaviors and prioritize user agency.

---

# Data Philosophy

BayWel does NOT aim to:
- exploit emotional vulnerability
- maximize engagement addiction
- monetize private reflections
- create psychological dependency

BayWel exists to support reflection and awareness.

---

# User Ownership

Users should control:
- whether memory is enabled
- what is stored
- what is deleted
- what is exported

Users should be able to:
- delete reflections
- disable memory
- revoke consent
- use anonymous mode

---

# Minimal Collection

Collect only what is necessary.

Prefer:
- structured metadata
- summaries
- anonymized analytics

Avoid unnecessary retention of:
- raw voice
- raw emotional journaling
- personally sensitive content

---

# Separation of Layers

## Raw Reflections
Potentially sensitive.

Should be:
- encrypted
- access-controlled
- deletable

---

## Structured Insights
Derived metadata.

Examples:
- themes
- categories
- emotional direction
- session statistics

This layer powers product improvement and personalization.

---

# Consent

Consent must be:
- explicit
- understandable
- granular
- reversible

No dark patterns.

No hidden consent.

---

# AI Transparency

Users should understand:
- AI is generating summaries
- AI is identifying themes
- AI is not a therapist
- AI may be imperfect

Avoid deceptive anthropomorphism.

---

# Wellness Boundaries

BayWel is:
- mindfulness
- reflection
- emotional awareness

BayWel is NOT:
- therapy
- psychiatric care
- diagnosis
- crisis intervention

The product should remain clearly non-clinical.

---

# Security Principles

- encrypted storage
- secure authentication
- Row Level Security
- minimal access permissions
- secure deletion support

---

# Long-Term Trust

Trust should become a competitive advantage.

As AI privacy concerns increase, BayWel should remain:
- calm
- transparent
- user-centered
- privacy-first
