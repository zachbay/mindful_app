# MVP Scope

# BayWel MVP Scope

# MVP Goal

Validate whether users engage with:
- physical mindfulness cards
- optional AI reflection
- longitudinal memory
- personalized insight tracking

without compromising the calm tactile experience.

---

# MVP Success Criteria

Validate:
1. QR scan engagement
2. Reflection completion rates
3. Repeat usage
4. Opt-in memory conversion
5. User emotional resonance
6. Creator content compatibility

---

# Included In MVP

## Physical Cards
Existing BayWel decks.

---

## QR Flow
Each card links to:
`/r/[deckId]/[cardId]`

---

## Reflection Experience
Users can:
- reflect anonymously
- enter text reflections
- optionally engage with AI prompts

---

## AI Reflection Support
AI provides:
- short reflective prompts
- summaries
- theme identification

No advanced therapy-like interaction.

---

## Opt-In Memory
After repeated usage:
Offer account creation.

Users can:
- save reflections
- track themes
- view summaries

---

## Authentication
- Google
- Apple
- magic link

No passwords.

---

## Basic Insight Dashboard
Simple dashboard showing:
- recurring themes
- reflection frequency
- recent summaries

No complicated analytics UI.

---

## Product Analytics
Track:
- card engagement
- completion rates
- repeat usage
- opt-in conversion

---

# Explicitly Excluded From MVP

Do NOT build initially:
- native mobile apps
- subscriptions
- payments
- gamification
- social networking
- public feeds
- creator monetization
- wearable integrations
- enterprise wellness dashboards
- voice AI
- advanced memory systems
- notification systems

---

# MVP Design Constraints

The product must remain:
- calm
- low-friction
- mobile-first
- emotionally safe
- privacy-forward

Avoid:
- SaaS dashboard aesthetics
- productivity app behavior
- excessive onboarding
- aggressive engagement tactics
